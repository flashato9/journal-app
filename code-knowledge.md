# .vscode

- Purpose
  - How does a project configure VS Code editor behavior and recommend essential development tools to ensure consistent development experience across all team members?
- Examples
  - `extensions.json` — recommends the Expo IDE extension when someone opens the project
  - `settings.json` — auto-fixes and organizes imports on save (but only if a formatter extension like Prettier or ESLint is installed)
  - Checked into git, so all developers get the same setup
- How settings work
  - `settings.json` contains code action instructions (e.g., `source.fixAll`, `source.organizeImports`) but they only work if a formatter extension is installed
  - The extension is what actually performs the formatting — the settings are just telling VSCode to trigger them
  - Example: if Prettier is installed, Prettier handles those code actions; if ESLint is installed, ESLint does it; if neither is installed, the settings do nothing
